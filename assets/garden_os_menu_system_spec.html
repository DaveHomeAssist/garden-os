Save summary: shows chapter, season, score, active carry-forwards, keepsakes, recipes. A full state snapshot the user can read.
Chapter history: list of completed chapters with score, key event that fired, keepsake earned (if any). Lets users review what happened and why their current state looks the way it does.
Keepsakes & recipes: amber badge shows count. Tapping opens the collection grid (earned items colored, unearned grayed). Replaces the need to hunt through chapter summaries.
"Restart this chapter" vs "Start new campaign": two separate destructive actions. Restart replays from the current chapter's planning phase with the same carry-forward state. New campaign wipes everything and goes to Ch 1.
Menu header context: chapter number, season name, and current score shown at a glance. This is the scoreboard that Connor said was missing — not big on screen, but always findable here.
Shared implementation notes
Entry point: ⚙ icon, top-right of toolbar in both tools. Consistent placement across the ecosystem (Trailkeeper, PLB follow the same convention).
Panel behavior: opens as a floating panel anchored to the gear icon, not a full-page modal. Overlay dims the grid but doesn't block it. Clicking outside closes it. Esc closes it.
Single HTML file constraint: menu is a hidden div toggled via data-open attribute. No new files. CSS handles show/hide via attribute selector.
Confirm dialogs: all destructive actions (clear, restart, new campaign) use the existing coach bubble primitive — coachShowBubble() with a confirm/cancel pair instead of native confirm(). Consistent with the coach layer.
Mobile: panel slides up from the bottom on small screens. Tab stops and focus trap for keyboard/screen reader users. Esc still dismisses.
Scoreboard gap (Season Engine): the persistent HUD issue from Connor's session is partially solved by the menu header — chapter + season + score is always one tap away, even if it's not always on screen.
