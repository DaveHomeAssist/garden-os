# Garden OS — Current Implementation Plan

Last verified: 2026-04-22

## 1. Project Summary

### Objective

Advance Garden OS from the recent stabilization, documentation, and repo-cleanup work into the next executable delivery cycle: finish planner Track Phase 5 scoring integration, close Story Mode Game Track Phase 4 against its published exit criteria, and prepare the cross-track gate for Game Track Phase 5 open-world work.

### Key outcomes

- Re-establish one trusted baseline for both active surfaces:
  - repo-root planner and static tools
  - `story-mode/` Vite + Three.js runtime
- Complete planner Phase 5 so numeric scoring and reasoning read from one selected-cell payload with no score drift
- Harden planner verification so the known `#autoFillBtn` automation flake stops masking regressions
- Close Game Track Phase 4 with explicit proof for inventory, skills, crafting, durability, and save round-trip behavior
- Enter Game Track Phase 5 with a reviewed scene, save, and shared-data preproduction package instead of starting code blindly

### Assumptions

- Repo-root HTML tools remain zero-backend and must not gain a build step.
- `story-mode/` remains the only bundled runtime and continues to use Vite, Vitest, and Three.js.
- Planner logic stays in `garden-planner-v4.html` until Phase 5 parity proves extraction is safe.
- Any change to shared save, crop, or scoring contracts follows `docs/TRACK_COORDINATION.md`.
- Archive handling, old assets, and other non-product cleanup are out of scope for this implementation cycle.

## 3. Execution Notes

### Critical sequencing constraints

- Do not start shared save or crop-schema changes before Phase 2 and Phase 4 are both stable enough to survive a rollback.
- Do not remove any planner fallback path until locked fixture parity is documented.
- Do not begin Game Track Phase 5 implementation until the cross-track review has explicitly approved save, scoring, and crop-data touch points.
- Keep repo-root planner work build-free; any extraction or cleanup must preserve the no-build execution model.

### Parallelizable work

- Phase 4 gap analysis and smoke-script drafting can run in parallel with Phase 2 once Phase 1 baseline is locked.
- Phase 5 preproduction document drafting can begin during late Phase 4, but no shared-contract code should land until Phase 4 is fully gated.
- Documentation updates can happen alongside each phase as long as they describe merged behavior, not planned behavior.

### Quick wins vs long-lead tasks

**Quick wins**

- Reconfirm Story Mode test/build baseline
- Re-run planner fixtures and archive the current outputs
- Fix the `#autoFillBtn` harness path
- Document the canonical season-state rule for planner Phase 6

**Long-lead tasks**

- Finishing Game Track Phase 4 if save or scoring edge cases remain
- Preparing open-world scene, save, and multi-bed contracts for Phase 5
- Any cross-track schema work touching crop, scoring, or save contracts
