PHASE 2 — SCHEMA ALIGNMENT (PLANNER → SPEC SHAPE)
════════════════════════════════════════════════════════════════════════════════

Milestone: Every planner crop has all fields the scoring algorithm needs.
Effort: Medium-Large (38 crops × 6 new/changed fields)
Risk: Medium (data authoring required — values must be botanically sane)

For each of the 38 planner crops, add or convert these fields:

  Field                    Current Planner        Target (Spec)
  ───────────────────────  ─────────────────────  ───────────────────────
  id                       (implicit from key)    Explicit string
  faction                  category (9 values)    faction (8 values)
  support                  trellisRequired        Boolean (true/false)
  tall                     height (enum)          Boolean
  water                    waterNeed (text)       Numeric (2 or 3)
  companions               companionTags (cats)   Crop ID array
  conflicts                conflictTags (cats)    Crop ID array
  chapterUnlock            MISSING                Integer (1-9)
  recipes                  MISSING                String array
  eventVulnerabilities     MISSING                String array
  seasonalMultipliers      MISSING                {spring, summer, fall}

Conversion rules:

  faction mapping:
    cucurbit  → climbers    (vining cucurbits)
    cucurbit  → fruiting    (bush cucurbits — zucchini already there)
    legume    → climbers    (pole_beans, peas)
    legume    → fast_cycles (bush_beans — short, quick harvest)
    greens    → greens      (spinach, chard) OR fast_cycles (lettuce, arugula)
    roots     → roots
    herbs     → herbs
    brassica  → brassicas
    companion → companions
    flowers   → companions

  tall mapping:
    height === 'tall'  → tall: true
    everything else    → tall: false

  water mapping:
    waterNeed === 'high'   → water: 3
    waterNeed === 'medium' → water: 2
    waterNeed === 'low'    → water: 2

  support mapping:
    trellisRequired === true → support: true
    everything else          → support: false

  companions/conflicts:
    Must be authored per-crop as specific crop IDs.
    Use horticultural companion planting references:
    - Carrots & Onions (mutual companions)
    - Tomatoes & Basil (mutual companions)
    - Beans & Onions (mutual conflict)
    - Brassicas & Tomatoes (mutual conflict)
    Each new crop needs 1-3 companions and 0-2 conflicts.

  chapterUnlock:
    Tier crops by complexity for the Season Engine progression.
    1 = starter (lettuce, radish, basil, spinach, arugula, marigold)
    2 = easy (bush_beans, peas, pole_beans, cherry_tom)
    3 = intermediate roots (carrot, onion, beet, scallion, turnip)
    4 = intermediate greens (bok_choy, mustard_greens, tatsoi, mizuna)
    5 = brassicas + herbs (broccoli, kale, dill, cilantro, parsley, chives)
    6 = fruiting (pepper, eggplant, compact_tomato, zucchini)
    7 = cucurbits (beit_cucumber, slicing_cucumber, bush_cucumber)
    8 = advanced brassicas (compact_cabbage, kohlrabi)
    9 = specialists (nasturtium, garlic, head_lettuce, chard)

  eventVulnerabilities:
    Valid tags (from EVENT_DECK.json):
      pest-target, pest-repellent, pest-decoy
      blight-prone
      frost-sensitive, frost-hardy
      heat-sensitive, heat-tolerant
      wind-sensitive
    Assign 1-3 tags per crop based on botanical reality.

  seasonalMultipliers:
    {spring, summer, fall} — each 0.0 to 1.0.
    Cool-season crops: spring ~1.0, summer ~0.3, fall ~0.9
    Warm-season crops: spring ~0.4, summer ~1.0, fall ~0.3
    All-season crops: spring ~0.7, summer ~0.8, fall ~0.7

The planner keeps its EXTRA fields (trellisZone, protectedZone, habit,
height, spacingPerSqFt, waterNeed, companionTags, conflictTags,
varieties, successionFriendly, replacementCategory, heatTolerance, color,
category) — these are planner-specific UI/UX fields. But the SPEC fields
must also be present so scoring functions can reference them.

Deliverable: All 38 planner crops have both the planner-specific fields
             AND the full spec schema fields. No field is missing.
